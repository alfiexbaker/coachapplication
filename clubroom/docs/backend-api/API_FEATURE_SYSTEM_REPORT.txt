Clubroom API Feature System Report
Generated: 2026-06-15
Purpose: map current app features to /v1 API authority, Prisma/Supabase tables, relationship shape, and known gaps for the live-data cutover goal.

Overall answer

The full app is not 100 percent complete yet.

Current working estimate: about 88 percent through the API authority cleanup. The current API boundary audit has zero findings and zero allowed baseline findings. The app is still not complete because several feature areas are partial or deliberately fail closed where no real /v1 API exists yet.

Primary runtime rule: in API mode, server-owned product data must come from named /v1 routes backed by Prisma/Supabase or fail closed. Mock/local data can remain only for explicit mock/demo mode or device/session-only state.

Status legend

- Implemented: API route exists and has a runtime-backed repository or Prisma path.
- Partial: important /v1 coverage exists, but at least one live flow is still missing or fail-closed.
- Fail-closed: app/API refuses non-mock live behavior until a proper backend authority exists.
- Missing API: no real /v1 authority yet; feature should not use mock/local live data.

- Recurring instance cancellation, recurring-series end, session-detail reassignment, and off-platform attendee adjustments need dedicated `/v1` authorities.
