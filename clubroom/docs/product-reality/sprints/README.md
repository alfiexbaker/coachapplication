## Rules

- Do not delete old sprint stacks until the new planning layer is approved.
- `archive/` contains superseded sprint drafts that should not drive current implementation.
- Retire stale findings before they become sprint scope.
- Separate:
  - implementation problems
  - UX honesty problems
  - product-definition problems
- Macro decisions should be discussed before code is written to cement them.

## Deliverable Standard

Every sprint in this folder must answer:

1. what user truth is being fixed
2. what misleading surface is being removed or made real
3. which acceptance criteria prove the product is more believable afterward
