# Sprint 15 - POC Demo Readiness And Golden Paths

## Objective

Turn the product from "good architecture plus many features" into a complete, demoable POC with coherent seeded stories.

## Why This Sprint Exists

A POC is only complete when the app can be shown cleanly across the core roles without relying on explanation to bridge broken narrative gaps.

## Scope

1. Define the golden paths for:
   - owner
   - admin
   - head coach
   - coach
   - parent
2. Seed the demo org, staff, families, athletes, sessions, and bookings around one coherent story.
3. Tighten entry points so each role lands in the right place.
4. Run the core flows and log rough edges, confusing states, and broken copy.
5. Do a targeted polish pass on the highest-risk demo-breakers.

## Acceptance Criteria

- the core roles can be demoed end-to-end with one coherent org story
- seeded data supports the story without manual setup tricks
- the roughest narrative and UI gaps in the golden paths are removed
- the team can state exactly what the POC proves and what it does not

## Verification

- run role-based golden-path walkthroughs
- targeted flow checks for owner, coach, parent
- update demo notes and known-gap list

## Output

At the end of this sprint, Clubroom is a complete POC rather than only a strong internal prototype.
