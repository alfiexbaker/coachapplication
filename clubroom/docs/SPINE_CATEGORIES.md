# Product Spines for Clubroom

Use these four spines to organize work, link features across sprints, and avoid inventing parallel flows when existing ones can be extended.

## 1) Community & Growth
- Build on the existing social feed, hashtags, and groups to drive discovery (share to groups, AMA threads, Q&A).
- Reuse content primitives (posts, attachments, highlights) instead of new post types; add controls like edit/delete/share as extensions.
- Connect organisation pages/teams to groups so announcements and resources live in one stream.

## 2) Booking, Availability & Revenue
- Extend current booking flows and availability surfaces before adding new schedulers.
- Keep formats unified (1:1, group, camps) by using shared service definitions, pricing, and capacity logic.
- Layer monetisation add-ons (video review, bundles, discounts) onto the same checkout and state machine rather than creating separate flows.

## 3) Development & Analytics
- Tie objectives, drills, and evidence vault items to existing booking/session records for continuity.
- Present parent-friendly dashboards (radar, timelines, next steps) as views over the same objective/attendance data instead of bespoke trackers.
- When adding analytics, reuse existing timelines and cards to keep history coherent.

## 4) Trust, Safety & Operations
- Anchor verification, safeguarding badges, and dispute flags on the current profile/admin structures.
- Manage coach organisations (invite/kick/roles) through shared identity and roster models, not new role systems.
- Keep messaging defaults, consent, and incident reporting aligned with the same permissions framework used elsewhere.

## How to apply these spines across sprints
- Map each story to a spine first; note any crossover (e.g., a group booking announcement touches Community + Booking).
- Prefer extending shared components (cards, lists, state machines) so new features inherit existing behaviors and styling.
- When proposing a new feature, ask which spine it strengthens and whether an existing flow can be upgraded instead.
